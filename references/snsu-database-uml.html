SNSU Tracker - Database Schema (UML)

Sunday Night Start Up Tracker - Complete Database Design

📋 Database Overview

  • Purpose: Track multi-step startup processes across production lines with team coordination
  • Key Features: Real-time status tracking, sign-offs, checklists, historical data, configurable workflows
  • Users: Operations, Maintenance, Sanitation, Finish Wine, WAM Lab, Warehouse teams
  • Scalability: Supports multiple line groups (24/5, 24/7, Spirits) with custom workflows

📊 Key Design Decisions

  • Flexible Groups: line_groups allows dynamic grouping (24/5, 24/7, Spirits, custom)
  • Configurable Workflows: process_steps linked to groups enables different workflows per group
  • Historical Tracking: runs table captures each startup event with timing data
  • Audit Trail: step_executions records every step with timestamps and sign-offs
  • Custom Checklists: Templates can be standard or custom per team/task combination
  • Real-time Status: Status fields (not-started, in-progress, completed) enable live tracking
  • Communication: Notes linked to lines and optionally to specific runs
  • User Management: Users table supports authentication and team assignment

🔍 Status Values

  • step_executions.status: 'not-started', 'in-progress', 'completed'
  • runs.status: 'active', 'completed', 'cancelled'

⚡ Performance Indexes

  • CREATE INDEX idx_lines_group ON lines(line_group_id)
  • CREATE INDEX idx_steps_group ON process_steps(group_id, step_order)
  • CREATE INDEX idx_runs_line ON runs(line_id, created_at DESC)
  • CREATE INDEX idx_executions_run ON step_executions(run_id, step_id)
  • CREATE INDEX idx_notes_line ON communication_notes(line_id, created_at DESC)
  • CREATE INDEX idx_checklist_template ON checklist_items(template_id, item_order)
